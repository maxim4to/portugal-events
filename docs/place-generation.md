# Генерация мест в каталог (place-generation)

Как наполнять каталог мест (`data/places/*.json`) с помощью навыка
`generate-places`. Навык создаёт **кандидатов** (`status: "candidate"`) — их
проверяет и утверждает человек. На сайте показываются только **утверждённые**
(`approved`) места.

## 1. Запуск

1. Откройте интерактивную сессию Claude Code в корне репозитория.
2. Запустите навык: наберите `/generate-places` или просто попросите на русском,
   например «сгенерируй места по Синтре» / «добавь пляжи в Сетубале».
3. Навык использует **веб-поиск** (описания, координаты) и **Wikimedia Commons**
   (фото со свободной лицензией). Работает партиями ~20–30 мест за запуск.

Навык добавляет записи с `status: "candidate"` и `collections: []`, ничего не
коммитит и не трогает схему.

## 2. Проверка кандидатов

```bash
git status
git diff data/places/
```

Что стоит проверить точечно на нескольких записях:

- **coords** — вбейте `lat, lon` в карту, убедитесь, что точка попадает в место.
- **description** — реальное ли место, нет ли выдумок; есть практический совет.
- **driveMinutesFromLisbon / visitDurationHours** — правдоподобны ли.
- **photo** — если есть, лицензия должна быть свободной (CC0 / CC BY / CC BY-SA /
  Public domain), `url` ведёт на `upload.wikimedia.org`, `sourceUrl` — на
  страницу `File:` в Commons. Если фото сомнительное — удалите весь блок `photo`.
- **tags** — только из словаря: `закат, бесплатно, с детьми, купание, виды,
  история, вино, сёрфинг, природа, фото, ресторан`.

Предпросмотр сайта:

```bash
npm run dev
```

Важно: **кандидаты на сайте не отображаются** — рендерятся только `approved`.
Чтобы посмотреть конкретного кандидата, временно поменяйте у него
`"status": "candidate"` на `"approved"`, посмотрите, затем верните обратно (или
оставьте, если утверждаете).

## 3. Утверждение (promotion)

Для записей, которые вам нравятся, поменяйте `"status": "candidate"` на
`"approved"`:

- Аккуратной заменой по файлу (find/replace `"status": "candidate"` →
  `"status": "approved"`) — но только если в файле все кандидаты вас устраивают.
- Или вручную по каждой записи, если утверждаете выборочно.

Затем перепроверьте и закоммитьте:

```bash
npm run validate
npm test
git add data/places/
git commit -m "data: approve places"
```

## Памятка

- На сайт попадают только `approved` места. `candidate` — это промежуточное
  состояние (staging) для ревью.
- Никогда не ослабляйте схему (`schema/index.ts`), чтобы данные прошли валидацию.
  Правьте данные, а не схему.
