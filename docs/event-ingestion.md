# Обновление афиши (события)

Афиша — динамическая часть сайта. События хранятся в `data/events/<YYYY-MM>.json`,
прошедшие уезжают в `data/events/archive/`. На сайте показываются только будущие
(`dateEnd >= сегодня`), отсортированные по дате.

## Как обновить список (основной сценарий)

1. Открой папку проекта в Claude Code (чат-агент) и попроси: **«актуализируй
   события»** (или «обнови афишу»). Сработает скилл `ingest-events`.
2. Агент прочитает `data/sources.json` (включённые источники), обойдёт их и
   веб-поиск, соберёт события на ближайшие ~3 месяца по всей Португалии,
   приведёт к схеме, отбросит дубли и прошедшие, запишет в `data/events/`.
3. Он прогонит `npm run validate` и `npm test` и покажет сводку: сколько добавил,
   какие источники не открылись.
4. **Ревью:** `git diff data/events/` — проверь несколько событий (даты, город,
   ссылка, цена). Что не нравится — удали/поправь вручную.
5. Закоммить и запушь в `main` — GitHub Actions пересоберёт и опубликует сайт.

## Источники

Список — в `data/sources.json`. Добавить источник = добавить объект:

```json
{
  "id": "kebab-slug",
  "name": "Человекочитаемое название",
  "type": "website",            // website | telegram | rss
  "url": "https://...",          // для telegram — публичный канал, читается как t.me/s/<канал>
  "hint": "что на странице искать",
  "enabled": true                 // false — временно отключить
}
```

Телеграм-каналы: указывай `type: "telegram"` и `url` вида
`https://t.me/<канал>` — агент читает публичный превью `t.me/s/<канал>`
(работает только для публичных каналов, последние ~20 постов).

## Продвижение / чистка

- Только валидные события проходят `npm run validate`, иначе сборка сайта упадёт
  до публикации — это защита от кривых данных.
- Прошедшие события агент сам переносит в `data/events/archive/`. Архив на сайте
  не показывается, но остаётся в истории.
- Хочешь убрать событие — удали его из файла месяца и перевалидируй.

## Заметки

- Автоматизации (cron) нет намеренно — обновление по требованию, когда откроешь
  проект и попросишь. При желании позже это переедет в облачные routines без
  переделки: скилл и данные останутся те же.
- Города матчатся с `data/geo/cities.json` для координат на карте — если событие
  в городе, которого там нет, добавь город с координатами.
